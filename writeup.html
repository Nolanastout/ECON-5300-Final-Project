Each driver has two BAC tests (bac1 and bac2). To create one running variable for the RDD, I use the minimum of the two tests (bac_min). I then create indicators for being at/above the two policy thresholds (0.08 and 0.15).

Quick near-cutoff comparison

As an initial descriptive check, I compare observations just below vs just above each cutoff within a narrow window (±0.02 BAC). This is not the final regression discontinuity estimate, but it provides an early sense of whether outcomes and co-variates look similar near the threshold.

w <- 0.02  # window size (±0.02)

summary_near <- function(cutoff) {
  dwi %>%
    filter(bac_min >= cutoff - w, bac_min <= cutoff + w) %>%
    mutate(above = as.integer(bac_min >= cutoff)) %>%
    group_by(above) %>%
    summarise(
      n = n(),
      recid_rate = mean(recidivism),
      male_share = mean(male),
      white_share = mean(white),
      mean_age = mean(aged),
      acc_share = mean(acc),
      .groups = "drop"
    ) %>%
    mutate(cutoff = cutoff) %>%
    relocate(cutoff, above)
}

bind_rows(

Checking Data for Lumping

Now that the data has been cleaned and the causal logic has been explained, we need to test whether the assumptions of the RDD design are present in the data. Before we can begin performing our RDD analysis, we need to make sure that there is no manipulation of BAC that influences whether the drivers end up on one side of the cutoff or the other. This will be achieved with a lumping test. Next, we need to ensure that the drivers just above and below the cutoff are materially similar to each other. We will achieve this by performing placebo regressions.

We need to check that there is no manipulation in the running variable around the cutoff point. To do this, we will graphically look at the data, by creating bins, and check to see if there is a jump at the cutoff in the number of observations.

#Check lumping at the 0.08 cutoff

Both charts show a relatively consistent shape of the data, which means that we can trust that there is no manipulation in the running variable. We can now proceed with our placebo testing.

Run Placebo Tests

The second part of reviewing the data is to run placebo tests. We do this because we need to ensure that the other key RDD assumption is true, meaning that the only thing that changes at the cutoff is the punishment. To perform this test, we will re-run the RDD regressions but swap out recidivism with our other variables (male, white, aged, acc). If this key assumption holds, we should not expect to see any jump at the cutoff for any of these variables. If we do see a jump at the cutoff in any of these variables, then that would suggest that our two groups (above and below the threshold BAC level) differ in other ways, thus making our comparison invalid. We will perform this check for both the 0.08 BAC cutoff and the 0.15 BAC cutoff.

Run Regressions

Now that we have checked the key assumptions in our data set, we can proceed with the main analysis. We want to determine whether crossing the BAC threshold, and receiving a harsh punishment, causes a decrease in recidivism. First, we will center the running variable around each cutoff point, create a treatment indicator for being above the threshold and then interact these two variables. The coefficient on the treatment indicator shows the jump in recidivism at the cutoff, which is our estimate of the effect of a harsher punishment. We will run three different variations for each cutoff: 1) a wider window of 0.05, 2) a narrower window of 0.02, and 3) a wide window with our additional variables added as controls. We will compare our results among these three variations to see if our results are stable.

Crossing the first cutoff of 0.08 reduces recidivism by about 1.9 percentage points, and crossing the 0.15 cutoff reduces recidivism by an additional 0.75 percentage points. Therefore, we can say that harsher punishments do reduce the likelihood of re-offending.
